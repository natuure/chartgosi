# 차트고시 Codex 개발 로드맵

## 1. 로드맵 개요

이 문서는 Codex가 차트고시를 단계적으로 구현하기 위한 실행 로드맵이다. Phase 1은 실제 MVP 구현이 가능한 수준으로 상세화하고, Phase 2~5는 MVP 구조를 깨지 않고 확장하는 방향으로 정의한다.

기준 스택:

- Frontend: Next.js, React, TailwindCSS, TradingView Lightweight Charts
- Backend: FastAPI
- DB: PostgreSQL
- Cache: Redis
- AI: OpenAI
- Batch: Python, Pandas

Vercel 배포를 진행하려면 별도로 Vercel CLI 설치가 필요하다.

## 2. Phase 1: MVP

### 목표

홈 진입부터 문제 풀이, 답안 제출, 결과 확인, 오답 저장, 랭킹 반영까지 핵심 루프를 완성한다.

### 구현 범위

- Next.js 앱 초기 구조
- FastAPI 서버 초기 구조
- PostgreSQL 스키마와 마이그레이션
- 기본 패턴 10개 seed 데이터
- 샘플 문제 seed 데이터
- 홈 화면
- 문제/게임 화면
- 결과 화면
- 패턴별 훈련장
- 오답노트 기본 목록
- 랭킹 기본 목록
- 구독 상태 조회용 placeholder

### 핵심 사용자 플로우

1. 사용자가 홈 화면에 진입한다.
2. 오늘의 문제 또는 패턴 훈련을 선택한다.
3. 서버가 문제와 차트 데이터를 반환한다.
4. 사용자가 `up`, `sideways`, `down` 중 하나를 선택한다.
5. 답안 제출 API가 `user_answers`를 저장한다.
6. 결과 화면이 실제 다음 5봉, 정답 여부, 해설, 선택 비율을 보여준다.
7. 오답이면 오답노트에서 다시 확인할 수 있다.
8. 랭킹 데이터가 갱신된다.

### API 우선 구현

- `GET /api/v1/patterns`
- `GET /api/v1/questions/today`
- `GET /api/v1/questions/{question_id}`
- `GET /api/v1/patterns/{pattern_id}/questions`
- `POST /api/v1/questions/{question_id}/answers`
- `GET /api/v1/answers/{answer_id}/result`
- `GET /api/v1/wrong-notes`
- `GET /api/v1/rankings`
- `GET /api/v1/stats/me`
- `GET /api/v1/subscriptions/me`

### 완료 기준

- 샘플 사용자로 로그인 또는 개발용 mock auth가 가능하다.
- 홈에서 오늘의 문제를 시작할 수 있다.
- TradingView Lightweight Charts로 차트와 다음 5봉 가림 영역을 표시한다.
- 답안 제출 후 정답 여부와 해설을 볼 수 있다.
- `user_answers`에 선택값, 확신도, 이유 태그, 소요 시간이 저장된다.
- 오답노트는 틀린 문제만 보여준다.
- 랭킹은 최소 점수순 정렬로 동작한다.

### 주요 리스크

- 실제 차트 데이터 확보 전까지 seed 데이터 품질이 사용자 경험을 좌우한다.
- 차트 가림 영역 UX가 명확하지 않으면 게임 룰이 헷갈릴 수 있다.
- 인증을 너무 늦게 붙이면 답안/랭킹 데이터 연결이 흔들릴 수 있다.

## 3. Phase 2: AI 분석

### 목표

축적된 답안 데이터를 기반으로 사용자 성향 분석과 맞춤 훈련 추천을 제공한다.

### 구현 범위

- AI 리포트 생성 API
- AI 리포트 화면
- 패턴별 정답률 분석
- 공격성, 추세추종, 반전 감지, 확신도 보정 점수
- AI 코멘트
- 맞춤 훈련 추천

### 구현 방향

- `user_answers`, `questions`, `patterns`를 조인해 분석 입력 데이터를 만든다.
- 최소 풀이 수 기준을 둔다. 기본값은 최근 30문제.
- AI 호출 결과는 `ai_reports`에 저장한다.
- 최신 리포트가 있으면 재사용하고, 사용자가 요청할 때 새로 생성한다.

### 완료 기준

- 최근 풀이 기반 AI 리포트가 생성된다.
- 리포트에는 종합 점수, 패턴별 성과, 성향 점수, 추천 훈련이 포함된다.
- 풀이 수가 부족하면 리포트 대신 추가 풀이 안내가 나온다.

### 주요 리스크

- AI 코멘트가 투자 조언처럼 보이지 않도록 학습 피드백 표현을 유지해야 한다.
- 분석 기준이 불명확하면 사용자가 결과를 신뢰하기 어렵다.

## 4. Phase 3: 자동 패턴 탐지

### 목표

10년치 OHLCV 데이터를 기반으로 패턴 후보를 탐지하고 문제 DB를 자동 생성한다.

### 구현 범위

- OHLCV 수집 배치
- 슬라이딩 윈도우 기반 패턴 탐지
- 룰 기반 점수 계산
- AI 검증
- 문제 후보 생성
- 관리자 검수 또는 자동 승인 플로우

### 구현 방향

- `market_assets`, `ohlcv_candles`, `pattern_detections`, `question_generation_jobs`를 추가한다.
- 룰 기반 점수는 패턴별로 독립 함수로 관리한다.
- AI 검증은 문제 해설 초안과 패턴 적합성 판단에 사용한다.
- 검증 완료된 후보만 `questions`로 승격한다.

### 완료 기준

- 최소 1개 시장과 10개 패턴에 대해 문제 후보를 자동 생성한다.
- 생성된 문제는 기존 문제 풀이 API에서 그대로 사용할 수 있다.
- 문제 품질 검수 상태를 추적할 수 있다.

### 주요 리스크

- 패턴 탐지 정확도가 낮으면 서비스 신뢰도가 떨어진다.
- 데이터 공급원 라이선스와 사용 조건을 확인해야 한다.

## 5. Phase 4: 구독 및 랭킹

### 목표

무료/프리미엄 차등 기능과 랭킹 경쟁 요소를 강화한다.

### 구현 범위

- 구독 결제 연동
- 무료 사용자 하루 10문제 제한
- 프리미엄 무제한 문제
- 프리미엄 AI 리포트
- 시즌 랭킹
- 랭킹 부정 사용 감지
- 이벤트/보상 구조

### 구현 방향

- `subscriptions`를 결제 제공자와 동기화한다.
- API 권한 체크에서 `users.plan`과 `subscriptions.status`를 함께 확인한다.
- 랭킹은 일간/주간/월간/전체 기간을 지원한다.
- 비정상적으로 짧은 답변 시간, 반복 재도전, 동일 문제 남용을 랭킹 계산에서 보정한다.

### 완료 기준

- 무료 사용자는 일일 제한이 적용된다.
- 프리미엄 사용자는 AI 분석과 무제한 풀이가 가능하다.
- 기간별 랭킹과 내 순위가 안정적으로 표시된다.

### 주요 리스크

- 결제 실패/취소/환불 상태 처리가 누락되면 권한 오류가 발생한다.
- 랭킹 부정 사용 대응이 늦으면 경쟁 콘텐츠 신뢰가 떨어진다.

## 6. Phase 5: B2B API

### 목표

증권사, 투자 교육 서비스, 기업 고객이 차트 퀴즈 엔진과 분석 데이터를 사용할 수 있게 한다.

### 구현 범위

- B2B API 키 발급
- 조직/팀 단위 사용자 관리
- 문제 조회 API
- 답안 제출 API
- 조직별 리포트
- 사용량 제한과 과금 기준
- API 문서와 샌드박스

### 구현 방향

- B2B는 일반 사용자 인증과 분리된 API key 기반 인증을 사용한다.
- 조직 단위 테이블은 Phase 5에서 추가한다.
- 기존 `questions`, `user_answers`, `ai_reports` 구조를 재사용하되, 조직 소유권과 외부 사용자 ID 매핑을 추가한다.

### 완료 기준

- 외부 서비스가 API로 문제를 가져오고 답안을 제출할 수 있다.
- 조직 관리자가 사용자 성과 요약 리포트를 볼 수 있다.
- 사용량 제한과 API 로그가 기록된다.

### 주요 리스크

- B2B 고객별 데이터 격리가 중요하다.
- API 응답 안정성과 버전 관리가 필요하다.

## 7. Codex 실행 순서

### Step 1: 문서 기반 프로젝트 구조 생성

- `frontend/` Next.js 앱 생성
- `backend/` FastAPI 앱 생성
- `infra/` 또는 `db/` 마이그레이션 구조 생성
- 공통 README 작성

### Step 2: DB와 seed 데이터

- 7개 핵심 테이블 마이그레이션 작성
- 10개 패턴 seed 작성
- 샘플 문제 seed 작성

### Step 3: Backend MVP API

- FastAPI 라우터 구성
- Pydantic schema 작성
- 문제 조회, 답안 제출, 결과 조회 구현
- 오답노트, 랭킹, 통계 API 구현

### Step 4: Frontend MVP UI

- 다크모드, 네온 블루/퍼플 디자인 토큰 구성
- 홈 화면 구현
- 게임 화면 구현
- 결과 화면 구현
- 패턴 훈련장, 오답노트, 랭킹 기본 구현

### Step 5: 검증

- API 단위 테스트
- DB seed 검증
- 홈 → 문제 풀이 → 제출 → 결과 → 오답노트/랭킹 플로우 수동 테스트
- 모바일 해상도 UI 확인

## 8. Acceptance Checklist

- [ ] PRD의 화면별 기능이 앱 라우트와 매핑된다.
- [ ] DB 설계의 7개 핵심 테이블이 마이그레이션으로 구현된다.
- [ ] 모든 핵심 API가 FastAPI에서 동작한다.
- [ ] 차트 화면이 실제 OHLCV JSONB를 기반으로 렌더링된다.
- [ ] 답안 제출 후 `user_answers`에 판단 데이터가 저장된다.
- [ ] 오답노트와 랭킹이 저장 데이터 기반으로 표시된다.
- [ ] 투자 조언이 아니라 학습 서비스라는 고지가 포함된다.
