# 차트고시 PRD

## 1. 제품 개요

**차트고시**는 사용자가 과거 차트만 보고 다음 5개의 봉을 예측하는 게임형 투자 학습 서비스다. 단순 정답 맞히기보다 사용자의 판단 데이터, 확신도, 선택 이유, 패턴별 성과를 축적해 투자 판단 습관과 약점을 분석하는 것이 핵심 가치다.

서비스는 실거래나 투자 추천을 제공하지 않는다. 모든 콘텐츠는 차트 학습, 투자 교육, 판단 훈련 목적이며, 실제 매수/매도 권유가 아니다.

### 핵심 슬로건

> 과거는 답을 알고 있다. 다음 5봉을 맞혀라.

### MVP 목표

- 과거 OHLCV 차트를 보여주고 다음 5봉을 가린다.
- 사용자가 `상승`, `횡보`, `하락` 중 하나를 선택한다.
- 제출 후 실제 다음 5봉, 정답 여부, 해설, 다른 사용자 선택 비율을 보여준다.
- 패턴별 문제 풀이, 오답노트, 랭킹에 필요한 데이터를 저장한다.
- AI 분석은 MVP에서 기본 리포트 구조와 데이터 수집 기반을 준비하고, Phase 2에서 고도화한다.

### MVP 성공 기준

- 사용자가 홈에서 문제를 시작해 결과 화면까지 완료할 수 있다.
- 최소 10개 패턴의 문제를 조회하고 풀이할 수 있다.
- 답안 제출 시 사용자 선택, 소요 시간, 확신도, 선택 이유, 정답 여부가 저장된다.
- 오답노트와 랭킹이 저장된 풀이 데이터를 기반으로 동작한다.
- AI 리포트 생성을 위한 원천 데이터가 누락 없이 축적된다.

## 2. 사용자와 권한

### 사용자 유형

| 유형 | 설명 | 주요 권한 |
| --- | --- | --- |
| 게스트 | 로그인 전 사용자 | 샘플 문제 풀이, 홈 탐색 |
| 무료 사용자 | 가입 사용자 | 하루 10문제, 랭킹, 기본 통계 |
| 프리미엄 사용자 | 유료 사용자 | 무제한 문제, AI 분석, 오답노트, 패턴 훈련 |
| B2B 사용자 | 기업/교육기관 사용자 | 퀴즈 엔진/API 사용, 조직 단위 리포트 |

### 기본 제약

- 무료 사용자는 하루 10문제 제한을 가진다.
- 프리미엄 기능은 구독 상태가 `active`일 때만 접근 가능하다.
- B2B API는 Phase 5에서 별도 인증과 사용량 제한을 둔다.

## 3. 화면별 기능 정의

### 3.1 시작/홈 화면

**목적**: 사용자가 오늘의 문제, 패턴 훈련, 오답노트, AI 리포트, 랭킹으로 진입하는 메인 허브.

**주요 UI**

- 앱명 `차트고시`
- 연속 퀴즈 일수
- 오늘의 문제 풀기 CTA
- 패턴별 훈련장 카드
- 오답노트 카드
- 10가지 패턴 카드
- 하단 탭: 홈, 랭킹, 통계, 이벤트, 더보기

**사용자 액션**

- 오늘의 문제 풀기
- 패턴 카드 선택
- 오답노트 이동
- 랭킹/통계/더보기 이동

**상태**

- 기본: 문제 풀기 가능
- 제한: 무료 사용자 일일 문제 수 초과
- 빈 상태: 추천 문제가 없거나 서버 오류

**성공 케이스**

- 오늘의 문제 CTA 클릭 시 문제 화면으로 이동한다.
- 패턴 카드 클릭 시 해당 패턴 문제 목록 또는 첫 문제로 이동한다.

**실패 케이스**

- 인증 만료 시 로그인 화면으로 유도한다.
- 문제 조회 실패 시 재시도 버튼을 보여준다.

### 3.2 오늘의 문제 화면

**목적**: 랜덤 또는 추천 문제 1개를 바로 풀이하게 한다.

**주요 UI**

- 문제 번호
- 난이도
- 패턴명
- 정답률
- 차트
- 다음 5봉 가림 영역
- 3지선다 선택 버튼
- 정답 제출 버튼

**사용자 액션**

- 선택지 선택: `up`, `sideways`, `down`
- 확신도 선택: MVP에서는 선택 입력을 저장할 수 있게 열어두고 UI는 선택 사항
- 선택 이유 태그 선택: 거래량, 이동평균선, 패턴, 추세, 감
- 답안 제출

**상태**

- 풀이 중
- 선택 완료
- 제출 중
- 제출 완료
- 시간 초과

**성공 케이스**

- 답안을 제출하면 `user_answers`에 저장되고 결과 화면으로 이동한다.

**실패 케이스**

- 선택지 없이 제출하면 제출 버튼을 비활성화한다.
- 중복 제출 요청은 마지막 정상 제출 1건만 인정한다.

### 3.3 게임 화면

**목적**: 연속 문제 풀이 경험을 제공한다.

**주요 UI**

- 상단: 뒤로가기, 생명, 제한시간, 진행도, 문제 번호, 정답률
- 중앙: 차트, 이동평균선, 거래량, 다음 5봉 가림 영역
- 하단: 3지선다 답안 버튼, 제출 버튼

**사용자 액션**

- 문제 풀이
- 즐겨찾기
- 제출
- 다음 문제 이동

**상태**

- 진행 중
- 생명 감소
- 세션 종료
- 네트워크 재시도

**성공 케이스**

- N문제 세션의 진행률과 점수가 갱신된다.

**실패 케이스**

- 제한시간 초과 시 오답 또는 미응답으로 저장한다.

### 3.4 결과 화면

**목적**: 실제 다음 5봉과 해설을 보여주고 학습 효과를 만든다.

**주요 UI**

- 실제 다음 5봉 공개
- 내 선택과 정답 비교
- AI 해설
- 다른 사용자 선택 비율
- 문제 정답률
- 다시 풀기, 다음 문제, 오답노트 저장

**사용자 액션**

- AI 해설 확인
- 다음 문제 풀기
- 오답노트 이동
- 재도전

**상태**

- 정답
- 오답
- 해설 로딩
- 사용자 통계 로딩

**성공 케이스**

- 결과 조회 시 정답, 해설, 사용자 선택 비율이 함께 표시된다.

**실패 케이스**

- AI 해설이 없으면 기본 규칙 기반 해설을 표시한다.

### 3.5 패턴별 훈련장

**목적**: 10개 기본 패턴별 문제를 반복 훈련한다.

**초기 패턴**

1. 컵앤핸들
2. W바닥
3. 박스권 돌파
4. 신고가 돌파
5. 눌림목
6. 삼각수렴
7. 플래그
8. 플랫베이스
9. 상승장악형
10. 거래량 급증

**사용자 액션**

- 패턴 선택
- 난이도 필터
- 훈련 시작

**성공 케이스**

- 선택한 패턴의 문제만 순차 출제한다.

**실패 케이스**

- 해당 패턴 문제가 부족하면 추천 패턴 또는 오늘의 문제로 안내한다.

### 3.6 오답노트

**목적**: 틀린 문제를 복습하고 취약 패턴을 확인한다.

**주요 UI**

- 오답 목록
- 패턴/난이도/시장 국면 필터
- 내 선택, 정답, 해설
- 재도전 버튼

**사용자 액션**

- 오답 상세 보기
- 해설 다시 보기
- 재도전

**성공 케이스**

- `user_answers.is_correct = false`인 답안을 최신순으로 보여준다.

**실패 케이스**

- 오답이 없으면 추천 훈련 문제를 보여준다.

### 3.7 AI 분석 리포트

**목적**: 단순 정답률이 아닌 투자 판단 성향과 약점을 요약한다.

**주요 UI**

- 종합 점수
- 상위 퍼센트
- 패턴별 정답률
- 성향 분석: 공격성, 추세추종, 반전 감지, 확신도 정확도
- AI 코멘트
- 맞춤 훈련 시작 CTA

**사용자 액션**

- 기간 선택
- 패턴 분석 탭 이동
- 오답 노트 이동
- 맞춤 훈련 시작

**성공 케이스**

- 최근 N문제 기준으로 분석 결과를 보여준다.

**실패 케이스**

- 풀이 수가 부족하면 최소 문제 수를 안내한다.

### 3.8 랭킹

**목적**: 점수, 정답률, 연속 풀이 기록을 경쟁 요소로 제공한다.

**주요 UI**

- 일간/주간/월간/전체 랭킹
- 내 순위
- 점수, 정답률, 풀이 수

**사용자 액션**

- 기간 전환
- 내 순위 보기

**성공 케이스**

- 랭킹 집계 데이터가 최신 기간 기준으로 표시된다.

**실패 케이스**

- 랭킹 데이터가 없으면 첫 참여 유도 문구를 보여준다.

### 3.9 통계

**목적**: 개인 풀이 성과와 추세를 보여준다.

**주요 UI**

- 누적 풀이 수
- 전체 정답률
- 패턴별 정답률
- 최근 점수 변화
- 평균 선택 시간

**사용자 액션**

- 기간 변경
- 패턴 필터

**성공 케이스**

- `user_answers` 기반 통계를 즉시 계산하거나 캐시된 결과를 표시한다.

**실패 케이스**

- 데이터 부족 시 샘플 안내와 문제 풀이 CTA를 보여준다.

### 3.10 구독/더보기

**목적**: 계정, 구독, 설정, 약관, 학습 고지로 접근한다.

**주요 UI**

- 내 정보
- 구독 상태
- 프리미엄 혜택
- 알림 설정
- 서비스 고지

**사용자 액션**

- 구독 시작/관리
- 로그아웃
- 약관 확인

**성공 케이스**

- 구독 상태가 `subscriptions`와 동기화된다.

**실패 케이스**

- 결제 실패 시 상태를 `past_due` 또는 `canceled`로 반영하고 재시도 CTA를 보여준다.

## 4. 사용자 플로우

### 4.1 첫 진입

1. 사용자가 홈 화면에 진입한다.
2. 게스트는 샘플 문제 또는 로그인 CTA를 본다.
3. 가입 사용자는 연속 퀴즈 일수, 오늘의 문제, 패턴 카드를 본다.
4. 오늘의 문제 풀기를 누르면 문제 조회 API를 호출한다.

### 4.2 문제 풀이

1. 서버가 문제와 과거 차트 데이터를 반환한다.
2. 클라이언트는 다음 5봉을 가린 차트를 렌더링한다.
3. 사용자는 상승/횡보/하락 중 하나를 선택한다.
4. 선택 시간, 확신도, 이유 태그를 함께 제출한다.
5. 서버는 정답 여부를 계산하고 답안 로그를 저장한다.

### 4.3 결과 확인

1. 서버는 실제 다음 5봉, 정답, AI 해설, 사용자 선택 비율을 반환한다.
2. 클라이언트는 결과 화면을 표시한다.
3. 사용자가 해설을 열면 `viewed_ai_explanation`을 갱신한다.
4. 다음 문제 또는 오답노트로 이동한다.

### 4.4 오답 복습

1. 사용자가 오답노트에 진입한다.
2. 서버는 사용자의 오답 목록을 반환한다.
3. 사용자는 해설을 확인하거나 재도전한다.
4. 재도전 답안은 `is_retry = true`로 저장한다.

### 4.5 패턴 훈련

1. 사용자가 패턴 카드를 선택한다.
2. 서버는 해당 패턴 문제를 난이도와 풀이 이력 기준으로 추천한다.
3. 사용자는 세션 단위로 여러 문제를 푼다.
4. 세션 종료 후 패턴별 정답률이 갱신된다.

### 4.6 AI 리포트 확인

1. 사용자가 AI 분석 리포트에 진입한다.
2. 서버는 최근 풀이 수가 충분한지 확인한다.
3. 충분하면 최신 리포트를 반환하거나 새 리포트를 생성한다.
4. 부족하면 추가 풀이 CTA를 보여준다.

### 4.7 랭킹 참여

1. 답안 제출 후 점수와 정답률이 갱신된다.
2. 배치 또는 즉시 집계를 통해 랭킹 레코드가 갱신된다.
3. 사용자는 기간별 랭킹에서 자신의 순위를 확인한다.

## 5. API 명세 초안

### 공통

- Base URL: `/api/v1`
- 인증: Bearer JWT
- 응답 형식: JSON
- 시간: ISO 8601 UTC
- 에러 형식:

```json
{
  "error": {
    "code": "QUESTION_NOT_FOUND",
    "message": "문제를 찾을 수 없습니다."
  }
}
```

### 5.1 인증

| Method | Endpoint | 목적 | 주요 테이블 |
| --- | --- | --- | --- |
| POST | `/auth/signup` | 회원가입 | `users` |
| POST | `/auth/login` | 로그인 | `users` |
| GET | `/me` | 내 정보 조회 | `users`, `subscriptions` |

### 5.2 문제 조회

| Method | Endpoint | 목적 | 주요 테이블 |
| --- | --- | --- | --- |
| GET | `/questions/today` | 오늘의 문제 조회 | `questions`, `patterns`, `user_answers` |
| GET | `/questions/{question_id}` | 문제 상세 조회 | `questions`, `patterns` |
| GET | `/patterns` | 패턴 목록 조회 | `patterns` |
| GET | `/patterns/{pattern_id}/questions` | 패턴별 문제 조회 | `questions`, `user_answers` |

**문제 응답 예시**

```json
{
  "id": "q_123",
  "pattern": {
    "id": "p_cup_handle",
    "name": "컵앤핸들"
  },
  "difficulty": "medium",
  "market_regime": "sideways",
  "base_date": "2024-06-21",
  "chart_data": [],
  "hidden_candles_count": 5,
  "answer_options": ["up", "sideways", "down"],
  "public_accuracy": 0.7
}
```

### 5.3 답안 제출과 결과

| Method | Endpoint | 목적 | 주요 테이블 |
| --- | --- | --- | --- |
| POST | `/questions/{question_id}/answers` | 답안 제출 | `user_answers`, `questions`, `rankings` |
| GET | `/answers/{answer_id}/result` | 제출 결과 조회 | `user_answers`, `questions`, `patterns` |
| PATCH | `/answers/{answer_id}/explanation-viewed` | AI 해설 확인 처리 | `user_answers` |

**답안 제출 요청 예시**

```json
{
```

## 6. 비기능 요구사항

- 차트 화면 초기 로딩은 2초 이내를 목표로 한다.
- 답안 제출 API는 중복 요청에 대해 멱등성을 보장해야 한다.
- 개인정보와 결제 정보는 최소 저장 원칙을 따른다.
- AI 리포트는 투자 조언이 아니라 학습 피드백임을 명확히 고지한다.
- 랭킹은 부정 사용 방지를 위해 비정상 풀이 속도와 반복 제출을 감지할 수 있어야 한다.
