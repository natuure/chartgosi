# 차트고시 변동성축소(VCP) 판정 기준

작성일: 2026-07-07

## 목적

변동성축소 패턴은 마크 미너비니(Mark Minervini)의 VCP(Volatility Contraction Pattern) 관점으로 정의한다.

VCP는 강한 상승 추세 이후 가격 조정폭과 거래량이 단계적으로 줄어들며 매물이 흡수되고, 마지막에는 좁은 변동폭 안에서 돌파 준비가 끝나는 구조다.

이 문서는 차트고시의 변동성축소 문제 생성, 패턴 설명, 스코어보드 표시, 향후 자동 패턴 탐지에 사용할 기준이다.

## 기본 관점

변동성축소는 **주봉 기준**으로 본다.

핵심 전제:

- 먼저 강한 선행 상승 추세가 있어야 한다.
- 이후 최소 2번, 최대 5번의 수축 구간이 나타나야 하며, 국소 고점은 최소 3개 이상이어야 한다.
- 각 수축은 이전 수축보다 가격 조정폭이 작아지는 것이 좋다.
- 수축이 진행될수록 거래량이 줄어드는 것이 좋다.
- 마지막 수축 구간은 매우 좁고 짧아야 한다.
- 문제 화면의 마지막 봉은 돌파 직전 또는 돌파 확인 봉으로 사용한다.
- 미래 돌파 후 흐름은 패턴 판정에 사용하지 않는다.

## 차트 표시 기준

변동성축소 문제 화면에서는 주봉 차트에 다음 이동평균선을 표시한다.

- MA10주
- MA30주
- MA40주

가격축은 로그 스케일을 사용한다.

## 기준 점수

- 85점 이상: 변동성축소 고신뢰 후보
- 75~84점: 변동성축소 후보
- 60~74점: 유사 패턴 가능성이 있어 보류
- 60점 미만: 변동성축소로 보기 어려움

## 변동성축소 스코어보드

| 항목 | 점수 | 기준 |
|---|---:|---|
| 선행 상승 추세 | 15 | VCP 형성 전 의미 있는 주봉 상승 추세가 있어야 함 |
| 국소 고점/수축 횟수 | 10 | 국소 고점 최소 3개 이상, 2~5번의 가격 수축 구간 확인 |
| 수축폭 감소 | 25 | 뒤로 갈수록 각 수축의 낙폭이 작아져야 함 |
| 거래량 감소 | 15 | 수축이 진행될수록 거래량이 줄어들면 가점 |
| 마지막 수축 품질 | 15 | 마지막 수축이 좁고 짧으며 매물 압력이 작아야 함 |
| 이동평균선 정배열 | 10 | 주봉 MA10 > MA30 > MA40 정배열 상태여야 함 |
| 피벗 돌파 품질 | 10 | 피벗 돌파봉 거래량이 직전 봉 대비 150% 이상이고, 윗꼬리가 짧을수록 가점 |

합계: 100점

## 현재 구현 기준

현재 변동성축소 탐색 로직은 기존의 단순 삼각수렴보다 다음을 더 중요하게 본다.

- 추세선 모양보다 **수축의 반복과 축소 순서**를 우선한다.
- 단순 고점 하락/저점 상승보다 **조정폭이 점점 작아지는지**를 우선한다.
- 수렴 마지막 구간에서 **거래량이 말랐는지**를 중요하게 본다.
- 일봉보다 주봉으로 보며, 너무 짧은 단기 수렴은 제외한다.

현재 구현은 마지막 봉 기준 최근 50주를 VCP 관찰 구간으로 사용한다.
수축 구간은 임의의 삼각형 추세선이 아니라, 선행 상승으로 만들어진 1차 국소 고점과 그 이후 같은 가격대에서 반복되는 국소 고점을 기준으로 자동 절단한다.

```text
VCP 관찰 구간 = 마지막 봉 포함 최근 50주
1차 국소 고점 = 선행 상승 구간에서 종가가 가장 높은 주봉
선행 상승 확인 구간 = 1차 국소 고점 전 2~5주
미래 5봉 = 정답 판정에만 사용하고 VCP 판정에는 사용하지 않음
```

## 필수 탈락 조건

| 항목 | 기준 |
|---|---|
| 선행 상승 부족 | 1차 국소 고점 전 2~5주 주봉 종가 기준 상승률이 30% 미만 |
| 국소 고점/수축 횟수 부족 또는 과다 | 국소 고점이 3개 미만이거나, 확인 가능한 수축이 2회 미만 또는 5회를 초과 |
| 수축폭 증가 | 뒤쪽 수축폭이 앞쪽 수축폭보다 크거나 같음 |
| 과도한 조정 | 1차 -45%, 2차 -33%, 3차 -25%, 4차 -15%, 5차 -8% 한도를 초과 |
| 이전 바닥 이탈 | 수축 도중 이전 수축 바닥 종가를 10% 이상 하향 이탈 |
| 이동평균선 정배열 아님 | 주봉 MA10 > MA30 > MA40 정배열이 아닌 상태 |
| 피벗 돌파 거래량 부족 | 피벗 돌파봉 거래량이 직전 봉 대비 150% 미만 |
| 피벗 돌파봉 윗꼬리 과다 | 피벗 돌파봉 윗꼬리 비율이 50% 이상 |

## 계산 방식 초안

### 1. 선행 상승 추세

VCP는 추세 지속형 패턴이므로, 패턴 형성 전 상승 추세가 먼저 있어야 한다.

초안 기준:

```text
선행 상승률 = 1차 국소 고점 주봉 종가 / 1차 국소 고점 전 2~5주 주봉 종가 - 1
```

현재 구현 기준:

- 45% 이상 상승: 15점
- 30~45% 상승: 10점
- 30% 미만: 후보 제외

최근 50주 관찰 구간 안에서, 선행 상승률 30% 이상을 만족하는 주봉 중 종가가 가장 높은 봉을 1차 수축 판단의 국소 고점으로 본다.
2~5주 중 가장 높은 종가 상승률을 사용한다.

### 2. 수축 구간

수축은 고점에서 저점까지의 조정 구간으로 본다.

```text
수축 낙폭 = 수축 시작 고점 종가 / 수축 저점 종가 - 1
```

VCP에서는 보통 다음과 같은 구조가 좋다.

```text
1차 수축 > 2차 수축 > 3차 수축
```

예시:

- 1차 수축: -25%
- 2차 수축: -15%
- 3차 수축: -8%

현재 구현은 주봉 종가 기준으로 다음 순서로 수축 구간을 자동 절단한다.

```text
1차 수축 시작 = 선행 상승에서 가장 종가가 높은 국소 고점 주봉
이후 국소 고점 = 1차 국소 고점 종가의 95~105% 범위 안에서, 앞뒤 5봉 중 종가가 가장 높은 주봉
수축 바닥 = 다음 국소 고점 전까지의 최저 종가 주봉
수축 낙폭 = 수축 시작 종가 / 수축 바닥 종가 - 1
```

수축 낙폭이 4% 미만이면 의미 있는 수축으로 보지 않는다.

수축별 최대 허용 낙폭:

| 수축 | 최대 낙폭 |
|---|---:|
| 1차 | -45% |
| 2차 | -33% |
| 3차 | -25% |
| 4차 | -15% |
| 5차 | -8% |

### 3. 수축폭 감소

각 수축폭이 순서대로 줄어드는지 본다.

초안 기준:

- 모든 수축이 순서대로 감소: 25점
- 대부분 감소하지만 한 구간이 비슷함: 18점
- 수축폭 감소가 약함: 10점
- 뒤쪽 수축이 앞쪽보다 크거나 같음: 후보 제외

현재 구현에서는 2~5개의 수축을 사용한다. 국소 고점이 3개 미만이면 후보에서 제외한다. 뒤쪽 수축이 앞쪽 수축보다 크거나 같으면 후보에서 제외한다.
또한 수축하는 도중 이전 수축 바닥 종가를 10% 이상 하향 이탈하면 후보에서 제외한다.

### 4. 거래량 감소

각 수축 구간의 평균 거래량이 줄어드는지 본다.

```text
수축 거래량 비율 = 뒤쪽 수축 평균 거래량 / 앞쪽 수축 평균 거래량
```

초안 기준:

- 수축마다 거래량이 감소: 15점
- 마지막 수축 거래량만 뚜렷하게 감소: 10점
- 거래량 감소가 약하거나 증가: 5점

현재 구현에서는 수축 구간 평균 거래량이 줄어들수록 점수를 높게 준다.
뒤쪽 수축 평균 거래량이 앞쪽 수축 평균 거래량보다 증가해도 후보에서 제외하지 않고 낮은 점수로 반영한다.

### 5. 마지막 수축 품질

마지막 수축은 짧고 좁을수록 좋다.

초안 기준:

- 마지막 수축 낙폭 5~10%: 고득점
- 마지막 수축 낙폭 10~15%: 보통
- 마지막 수축 낙폭 15% 초과: 감점
- 마지막 수축 기간이 너무 길어지면 감점

현재 구현에서는 마지막 수축의 순번에 따라 허용폭을 다르게 본다.
4차 수축은 최대 -15%, 5차 수축은 최대 -8%까지만 허용한다.

### 6. 이동평균선 정배열

주봉 MA10/30/40을 사용한다.

초안 기준:

- MA10 > MA30 > MA40: 후보 유지
- MA10 > MA30 > MA40 정배열이 아니면 후보 제외

현재 구현에서는 마지막 봉 기준 MA10 > MA30 > MA40 정배열을 필수 조건으로 사용한다.

### 7. 피벗 돌파 품질

마지막 봉 종가가 국소 고점들을 직선으로 이은 피벗선을 돌파했는지 본다.
피벗 돌파는 가격만 보지 않고 거래량과 윗꼬리 품질까지 함께 본다.

```text
피벗 가격 = 변동성축소 구간의 국소 고점들을 직선으로 이은 선의 마지막 봉 위치 가격
피벗 돌파율 = 마지막 봉 종가 / 피벗 가격 - 1
피벗 돌파 거래량 비율 = 피벗 돌파봉 거래량 / 직전 봉 거래량
피벗 돌파봉 윗꼬리 비율(양봉) = (고가 - 종가) / (고가 - 저가)
피벗 돌파봉 윗꼬리 비율(음봉) = (고가 - 시가) / (고가 - 저가)
```

초안 기준:

- 피벗선을 0~3% 이내로 돌파: 고득점
- 피벗선을 돌파했지만 3~8% 범위: 부분 점수
- 피벗선을 돌파하지 못했거나 8% 초과 과열 돌파: 후보 제외
- 피벗 돌파봉 거래량이 직전 봉 대비 150% 미만: 후보 제외
- 피벗 돌파봉 윗꼬리 비율이 50% 이상: 후보 제외
- 피벗 돌파봉 윗꼬리 비율이 0%에 가까울수록 고득점, 0~50% 범위에서 차등 점수

현재 구현에서는 마지막 봉 종가가 피벗선을 돌파해야 후보로 본다.
또한 피벗 돌파봉 거래량이 직전 봉 대비 150% 이상이어야 하며, 윗꼬리 비율이 50% 미만이어야 한다.

## 문제 화면 표시 근거

변동성축소 문제에서는 다음 근거를 표시한다.

- 선행 상승률
- 수축 횟수
- 각 수축 낙폭
- 수축폭 감소 여부
- 수축별 거래량 변화
- 마지막 수축 낙폭
- MA10/30/40 정배열 여부
- 피벗 가격과 돌파율
- 피벗 돌파 거래량/직전 봉 거래량
- 피벗 돌파봉 윗꼬리 비율

## 현재 한계와 향후 조정 후보

- 수축 구간 자동 절단은 1차 국소 고점과 95~105% 가격대의 후속 국소 고점을 기준으로 구현했다.
- 수축폭은 고가/저가가 아니라 종가 기준으로 계산한다.
- 피벗 가격은 국소 고점들을 직선으로 이은 선의 돌파 가격으로 구현했다.
- 일시적인 뉴스성 급등락은 VCP 구조를 왜곡할 수 있다.
- 주봉 기준이므로 문제 수가 일봉 패턴보다 적게 나올 수 있다.

## 앱 반영 상태

- 패턴 표시명은 `변동성축소`로 사용한다.
- 기존 DB/API 호환을 위해 slug는 당분간 `triangle`을 유지한다.
- 변동성축소 문제는 주봉 차트 기준으로 생성한다.
- 현재 생성 스크립트에서는 `triangle` slug를 주봉 VCP 로직으로 평가한다.
- 실제 탐색 로직은 `backend/scripts/find_real_remaining_pattern_questions.py`의 `evaluate_triangle()`에 구현되어 있다.
